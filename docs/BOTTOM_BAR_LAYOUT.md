## Zone Layout

Canvas 9:16 (1080x1920, 720x1280, 480x854) — all proportional.

```
┌─────────────────┐  ← y=0
│  HEADER ZONE    │  20%  ← header overlay image (logo, thumbnail, etc.)
├─────────────────┤
│                 │
│  VIDEO ZONE     │  70%  ← source video cropped to fill width.
│                 │       bottom edge TOUCHES top edge of bottom zone.
│                 │       (video does NOT overlap the bottom zone)
│                 │
├─────────────────┤  ← y = 80% canvasH
│  BOTTOM ZONE    │  10%  ← opaque accent bar + title text.
│  (PART 1)       │       border + title + background
└─────────────────┘  ← y = 100% canvasH
```

| Zone    | Height | Content                         |
|---------|--------|----------------------------------|
| Header  | 20%    | Header overlay image             |
| Video   | 70%    | Source video (16:9 cropped)    |
| Bottom  | 10%    | Opaque bar + "PART 1" text     |

**Key behavior:** Video zone bottom edge **touches** the top edge of the bottom zone. Video is NOT covered/overlapped by the bottom zone.

## Zone Constants

Defined in `electron/services/ffmpeg.ts`:

```typescript
// Zone heights as % of canvasH
export const HEADER_PCT = 0.20   // 20% — header overlay zone
export const BOTTOM_PCT = 0.10   // 10% — bottom bar zone
export const VIDEO_PCT  = 1 - HEADER_PCT - BOTTOM_PCT  // 70%

// For any canvas height:
//   headerH  = Math.floor(canvasH * HEADER_PCT)
//   bottomH  = Math.floor(canvasH * BOTTOM_PCT)
//   videoH   = canvasH - headerH - bottomH
//   bottomBarY = headerH + videoH  (= canvasH - bottomH)
```

## Bottom Bar PNG

Generated by `preRenderOverlays()` in `electron/services/ffmpeg.ts`.

- **Dimensions:** `canvasW × bottomH` (NOT full canvas — exact bar size)
- **Content:** Opaque accent color (`#00B4FF`) background + white text centered
- **Alpha fix:** LockBits forces A=255 on ALL pixels (fixes anti-aliasing FillRectangle artifacts)
- **Overlay position:** `y = headerH + videoH` (= `canvasH - bottomH`)

### PowerShell Script Logic

```powershell
# Bitmap: canvasW × bottomH (exact bar size)
$bmp = New-Object System.Drawing.Bitmap($canvasW, $bottomH)

# Fill with opaque accent color
$brush = New-Object System.Drawing.SolidBrush([System.Drawing.Color]::FromArgb(255, 0, 180, 255))
$g.FillRectangle($brush, 0, 0, $canvasW, $bottomH)

# Draw text centered
$fontSize = [int]($bottomH * 0.45)
$g.DrawString($text, $font, (New-Object System.Drawing.SolidBrush([System.Drawing.Color]::White)), $rect, $sf)

# CRITICAL: Force all pixels to A=255 (fix anti-aliasing)
$bd = $bmp.LockBits($rect, [System.Drawing.Imaging.ImageLockMode]::ReadWrite, ...)
for ($i = 3; $i -lt $bytes.Length; $i += 4) { $bytes[$i] = 255 }
$bmp.UnlockBits($bd)
```

**Why exact bar size (not full canvas)?**
- Full-canvas PNG with transparent bg → FFmpeg overlay blends transparent pixels → bar appears semi-opaque
- Exact bar size → fully opaque bar overlays directly at bottomBarY → correct color

## FFmpeg Filter Chain — SHORT Mode

```
Input [0]: source video (16:9)
Input [1]: background (blur/solid/image)
Input [2]: header overlay image
Input [3]: bottom bar PNG (canvasW × bottomH)

Filter Complex:
  [0:v] fps=30,setpts=PTS-STARTPTS,
        scale=-2:videoH,               # scale to target video height
        crop=canvasW:videoH:cropX:0    # crop center columns
        [vid]

  [1:v] scale=canvasW:canvasH:force_original_aspect_ratio=increase,
        crop=canvasW:canvasH:(ow-iw)/2:(oh-ih)/2,setsar=1
        [bg]

  [2:v] scale=canvasW:headerH:force_original_aspect_ratio=increase,
        crop=canvasW:headerH:(ow-iw)/2:(oh-ih)/2
        [hd]

  [bg][vid] overlay=0:headerH         # video over bg at headerH
        [vz]

  [vz][hd] overlay=0:0               # header on top
        [fh]

  [3:v] null                          # bottom bar PNG
        [bb]

  [fh][bb] overlay=0:bottomBarY      # bottom bar at (headerH + videoH)
        [final]
```

### Video Crop Math

Source 16:9 → canvas 9:16:
- `scale=-2:videoH` → source scaled to target height, width auto
- `cropX = (scaledWidth - canvasW) / 2` → crop center columns
- Result: video fills canvas width, BG shows through header + bottom zones

### Layer Z-Order (bottom to top)

1. **Background** — fills entire canvas
2. **Video** — at y=headerH, covers video zone
3. **Header overlay** — at y=0, on top
4. **Bottom bar** — at y=bottomBarY, opaque bar covers BG area

## Core Render Script

**File:** `scripts/render-core.ps1`

```powershell
.\render-core.ps1 -Video "path" -Blur "path" -Header "path" -Output "path"
.\render-core.ps1 -Video "src.mp4" -Blur "blur.jpg" -Header "hdr.png" -Output "out.mp4" -Text "PART 1" -Duration 30
.\render-core.ps1 -Video "src.mp4" -Blur "blur.jpg" -Header "hdr.png" -Output "out.mp4" -CanvasW 720 -CanvasH 1280
```

Params:
- `-Video` (required) — source video path
- `-Blur` (required) — blur/thumbnail background
- `-Header` (required) — header overlay image
- `-Output` (required) — output file path
- `-Text` (default: "PART 1") — bottom bar text
- `-Duration` (default: 30) — render duration in seconds
- `-CanvasW` (default: 1080) — canvas width
- `-CanvasH` (default: 1920) — canvas height
- `-Preset` (default: "ultrafast") — FFmpeg encoding preset
- `-CRF` (default: 22) — FFmpeg CRF quality

## Landscape Mode

Landscape mode (16:9 canvas) uses a different layout:

```
┌────────────────────────────────────┐
│  HEADER ZONE (thumbnail bg shows)  │  ← header overlay
├────────────────────────────────────┤
│                                    │
│  VIDEO ZONE (vidHeightPct%)       │  ← video centered/covering
│                                    │
├────────────────────────────────────┤
│  TITLE ZONE (remaining)           │  ← title overlay or drawtext
└────────────────────────────────────┘
```

Landscape does NOT use the bottom bar system. It uses the traditional title overlay approach.
